# MVP 功能规格

> 小程序实现以 `miniprogram-mvp/` 为准；类型常量见 `utils/constants.js`（含 `resourceTypeHints` / `demandTypeHints`）。  
> **完整功能基线记录**见 [12_功能清单.md](12_功能清单.md)（页面、流程、状态机、云 API 全量口径）。

## 1. 平台范围

面向算力、IDC、机房建设、数据中心运营与渠道代理的 B2B 供需撮合。第一版不做在线支付与自动调度，重点做结构化发布、认证、审核、匹配与人工撮合。

## 2. 用户与认证

- 访客：浏览公开展示池（不含附件与发布方联系方式）
- 普通用户：注册登录、名片/执照认证、发布、对接
- 运营账号：`staffRole` 审核、代发、代撮合（不以个人身份发布）

认证等级：名片认证 → 营业执照认证（详见 [10_权限规则.md](10_权限规则.md)，v1.13：对接登录即可，发布需名片）。

## 3. 资源发布

### 资源类型（MECE · 统一四字）

类型名只描述**品类**，不用销售/供应/采购/融资等交易动词；资源与需求可对齐。

| 类型 | 说明 | 边界 |
| --- | --- | --- |
| 算力整机 | 完整服务器/算力设备 | 买断/交付；按租归算力租赁 |
| 硬件配件 | 模组、电源、线缆等 | 零部件 |
| 算力租赁 | GPU、机柜、带宽等按租使用 | 单客户租用 |
| 运维维保 | 巡检、备件、驻场维护 | 设备级维护 |
| 机房建设 | 设计、施工、液冷工程 | 一次性建设 |
| 托管运营 | 数据中心整站托管/代运营 | 设施级运营 |
| 资金支持 | 项目融资、设备分期等金融合作 | 不含具体设备交付 |
| 综合资源 | 以上未覆盖的其它合作 | 兜底；描述须写清 |

发布/筛选时展示 `resourceTypeHints` 一行说明，减少误选。旧名称（整机、其他资源、GPU算力等）自动映射。

### 发布流程

1. 用户填写资源信息
2. 平台字段完整性检查
3. 后台人工审核
4. 审核通过后进入资源池
5. 系统或撮合经理推荐匹配需求

## 4. 需求发布

### 需求类型

与资源池 **完全相同的 8 类**：算力整机、硬件配件、算力租赁、运维维保、机房建设、托管运营、资金支持、综合资源。

旧名称（训练算力、推理部署、IDC 托管、代理合作等）自动映射至新类型。

### 发布流程

1. 用户填写需求信息
2. 平台判断完整度
3. 审核后进入需求池
4. 推荐匹配资源
5. 申请对接或交换联系方式

## 5. 商机匹配

- 按产业链类型互补、地域、标签、信息完整度等计算匹配分
- 详情页对发布者展示智能推荐
- 完整规则见 `utils/constants.js` 中 `resourceTypeMap` / `demandTypeMap`

## 6. 对接与审核

- 需求方 → 资源方：资源详情申请对接
- 资源方 → 需求方：需求详情勾选自有资源匹配
- 代发商机对接需运营审批
- 发布方信息默认隐藏，按权限逐步放开（见权限规则文档）
