# 算力贸易平台 · 权限规则（v1.13）

> 本文档是小程序商机（资源 / 需求）权限的**唯一口径**。开发与产品文案均以此为准。  
> 代码实现见 `miniprogram-mvp/utils/permissions.js`（详情页上下文）、`miniprogram-mvp/utils/certGate.js`（认证门禁）、`miniprogram-mvp/utils/data/_core.js`（对接/关闭/池隐藏等）。  
> 功能范围见 [12_功能清单.md](12_功能清单.md)。

---

## 一、设计原则

1. **身份、认证、关系三层分离**  
   - **账号身份**：你是谁（访客 / 普通用户 / 运营）  
   - **认证等级**：你有多可信（未认证 / 名片 / 执照）  
   - **商机关系**：你和某条资源或需求是什么关系（路人 / 发布方 / 对接方 / 代发运营）

2. **一个账号一种身份**  
   - 有 `staffRole` → 运营账号，**不能**以个人身份发布、对接、认证  
   - 无 `staffRole` → 普通用户  
   - **不可在同一账号内切换**两种身份

3. **发布方 ≠ 代发运营**  
   - **发布方**：商机真正的主人（普通用户，或代发商机的客户手机号持有人）  
   - **代发运营**：替客户录入、撮合的运营专员，**不是**发布方  
   - **关闭**：普通用户发布方仅可关闭**自己的**商机；代发运营可关闭**其负责代发**的商机（`canStaffManageProxyListing`），**不可**关闭他人自行发布的商机  
   - 分享：发布方与负责代发运营均可分享；代发分享链接对外隐藏客户信息

4. **信息分级展示**  
   - 公开展示字段 → 人人可见（含访客）  
   - 发布方身份 → 默认隐藏，按关系逐步放开  
   - 资源附件 → 比正文更严，需执照认证或特殊关系  
   - 需求附件 → 不公开展示（仅发布方、运营审核可见）

5. **对接方向固定**  
   - **需求方 → 资源方**：在资源详情「申请对接」  
   - **资源方 → 需求方**：在需求详情「我有匹配资源」（勾选自有资源）  
   - 运营代发走代发管理 / 详情代发视图，不以个人身份操作

## 二、账号身份（登录后固定）

| 身份 | 判定条件 | 一句话 |
| --- | --- | --- |
| **访客** | 未登录 | 只能看、不能动 |
| **普通用户** | 已登录，无 `staffRole` | 可认证、发布、对接 |
| **运营专员** | `staffRole = manager` | 审核、代发、代撮合 |
| **平台管理员** | `staffRole = admin` | 专员权限 + **平台治理**（强制下架商机、禁用/解禁普通用户账号） |

运营账号**不能**：个人发布资源/需求、个人申请对接、个人做企业认证、在公开展示池以个人身份对接/匹配。

### 2.1 账号状态与微信绑定

| 规则 | 说明 |
| --- | --- |
| **账号禁用** | `users.accountStatus = disabled` 时无法登录、无法会话校验通过；其商机由平台治理流程下架 |
| **微信 openid 绑定** | 同一 openid 绑定手机号后，`allowRegister = false`（不可再注册新号，仅可登录）；登录 Tab 可锁定已绑定手机号（`phoneLocked`） |
| **多账号** | 同一微信可注册**不同**手机号账号；资料与记录以**手机号**隔离 |
| **退出登录** | `logoutUser()` 清空本机缓存；**当前 UI 无退出入口**（会话失效时自动触发） |

## 三、认证等级（仅普通用户，逐步解锁）

> **操作门禁**以 `certGate.js` 为准；**「我的」页能力标签**按 `getUserAccessLevel()` 展示进度，两者口径见 §3.2。

### 3.1 操作门禁（硬规则）

| 等级 | 条件 | 解锁能力 |
| --- | --- | --- |
| **已注册** | 手机号 + 密码登录 | 浏览、收藏、维护资料、**申请对接 / 发起匹配**（登录即可，对接表单自带企业信息，**不弹**名片认证） |
| **名片认证** | 名片审核通过 | 已注册全部能力 + **提交需求 / 发布资源**（含整机、机房类发布，`submit_listing`） |
| **执照认证** | 营业执照审核通过 | 名片全部能力 + **查看他人资源附件** + 执照信任标签 + 列表优先推荐 |

认证是**叠加**关系：有执照视为同时有名片能力。

**人工撮合**（`type=match`）：已登录即可提交，不要求名片认证。

### 3.2 「我的」页能力标签（展示用）

`permissions.buildPermissionRows(accessLevel)` 按认证进度展示 9 项能力；其中「申请对接 / 发起匹配」在 UI 上归入 `standard`（名片）档展示为已解锁，但**实际操作**已注册用户即可对接（见 §3.1）。以操作门禁为准。

| accessLevel | 条件 | 标签展示 |
| --- | --- | --- |
| `guest` | 未登录 | 仅 2 项浏览 |
| `basic` | 已登录未认证 | 浏览 + 待解锁认证/发布/对接标签 |
| `standard` | 名片认证 | 发布、对接等核心项显示已解锁 |
| `verified` | 执照认证 | 含附件、优先推荐 |

## 四、与单条商机的关系（动态）

对每一条资源（`URES-`）或需求（`UDEM-`），系统按当前登录用户判定关系：

| 关系 | 判定 | 说明 |
| --- | --- | --- |
| **路人** | 无下列关系 | 浏览公开展示信息 |
| **发布方** | 手机号 / openid 与商机归属一致 | 主人：管理、关闭、看完整发布方信息 |
| **已授权对接方** | 存在有效对接，且状态 ≠「待平台审核」 | 可先见发布方**脱敏企业名** |
| **代发运营** | 该商机 `publishedByStaff` 且当前运营为代发专员 | 运营视图：客户信息、撮合、对接记录 |

> **有效对接**：对接记录未关闭、未流失。  
> **已授权**：平台对接审核已通过，或无需平台审核已进入后续流程。

同一人可同时是 A 商机的发布方、B 商机的路人。

## 五、资源商机 · 信息可见性

### 5.1 公开展示字段（人人可见，含访客）

标题、类型、地区、规模、价格/预算区间、规格列表、摘要、标签、发布时间、认证标签（执照/名片）、是否平台代发等**商机正文**。

不含：发布方企业全称、联系人、手机号、资源附件文件。

### 5.2 发布方信息

| 查看者 | 企业名称 | 联系人 / 手机 |
| --- | --- | --- |
| **访客（未登录）** | ❌ 不展示 | ❌ |
| 已登录路人 | ❌ 不展示 | ❌ |
| 发布方本人 | ✅ 完整 | ✅ 完整 |
| 代发运营（该商机，非分享落地） | ✅ 完整（**客户信息**卡片，非发布方区） | ✅ 完整 |
| **平台管理员**（`staffRole = admin`，非分享落地） | 普通用户发布的资源/需求：✅ 完整（云端拉取） | ✅ 完整 |
| **运营专员**（`manager`） | ❌ 不展示（普通用户发布） | ❌ |
| 已授权对接方 | ✅ **脱敏**企业名 | ❌ 交换前不展示 |
| 双方同意交换联系方式后 | ✅ 脱敏企业名 | ✅ 完整 |

列表卡片**不展示**发布方企业名（避免从脱敏名推断）。

**代发商机分享**：运营分享出去的详情链接（`from=share`），路人/访客**不得**看到代发客户企业、联系人、手机号；公开展示池接口与本地缓存同步时亦剥离上述字段。负责该代发商机的运营（`canStaffManageProxyListing`）在**非分享落地**的详情页可查看客户信息（含代发管理、提交落地、记录跳转等任意入口）；分享链接与其它非负责运营一律不展示运营视图。

### 5.3 资源附件

| 查看者 | 可否查看 |
| --- | --- |
| 访客 | ❌ |
| 路人（已登录，无执照） | ❌，引导执照认证 |
| 路人（已登录，有执照） | ✅ |
| 发布方本人 | ✅ |
| 代发运营（该商机） | ✅ |
| 已授权对接方 | 与「路人+执照」相同：有执照可看 |

需求附件：**永不公开展示**（仅发布方本人、运营审核可见）。

## 六、资源商机 · 操作权限

### 6.0 操作总表

| 操作 | 访客 | 已注册未认证 | 名片认证 | 发布方本人 | 代发运营（负责） | 运营（路人） |
| --- | --- | --- | --- | --- | --- | --- |
| 浏览资源池 / 详情 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 收藏 | 引导登录 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 申请对接（需求→资源） | 引导登录 | ✅ | ✅ | ❌ 自己的 | ❌ | ❌ → 代发流程 |
| 发起匹配（资源→需求） | 引导登录 | ✅ | ✅ | ❌ 自己的 | ❌ | ❌ → 代发流程 |
| 发布需求 / 资源 | 引导登录 | 引导认证 | ✅ | ✅ | ❌ → 代发 | ❌ → 代发 |
| 详情勾选批量匹配 | — | — | — | ✅ | ✅ 代发商机 | ❌ |
| 关闭商机 | ❌ | ❌ | ❌ | ✅ | ✅ 仅负责代发 | ❌ |
| 分享菜单（好友/朋友圈） | ❌ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 详情「邀请好友对接/匹配」 | ✅ | ✅ | ✅ | ❌ | ❌ | ❌ |

> 对接/匹配门禁：`ensureConnectAccess()` → 登录 + 非运营。发布门禁：`ensureSubmitListingAccess()` → 名片或执照（`certGate.submit_listing`）。

### 6.1 公开展示池 · 列表快捷操作

| 池 | 列表操作 | 条件 |
| --- | --- | --- |
| **资源池** | 「申请对接」 | 已登录普通用户、非发布方、非运营；不进详情 |
| **需求池** | 「发起匹配」 | 同上 |
| **两池** | 「已申请 ✓」 | 已有进行中对接，跳转记录 |
| **两池** | 发布 FAB `+` | 普通用户进发布表单；运营引导代发 |

运营浏览公池：卡片底部**仅「查看详情」**，无个人对接/匹配按钮。

### 6.2 公开展示池隐藏规则

- **自己发布的资源/需求**，在资源池/需求池列表中**不展示给自己**（防止自对接、保持池子干净）  
- 代发商机审核通过后，对他人正常展示；**客户本人**登录仍视为发布方，列表中隐藏  
- 运营浏览公开展示池时，**不因代发身份**额外隐藏（与客户视角一致，均可见他人商机）

### 6.3 待审核 / 已关闭

| 状态 | 路人 | 发布方 | 代发运营 |
| --- | --- | --- | --- |
| 待平台审核 | 不可见 | 可见预览 | 可见 |
| 已关闭 | 不可见 | 记录可查 | 可见历史 |

### 6.4 详情页 UI 分区（`buildListingPermissionContext`）

| 角色 | 详情页展示 |
| --- | --- |
| 路人（已登录） | 底部主按钮（申请对接 / 发起匹配）；**无**内嵌勾选区 |
| 发布方 | 商机管理区 + 勾选匹配；**无**底部个人申请栏 |
| 代发运营（负责） | 商机管理区 + 代发勾选匹配（`staffResource` / `staffDemand`） |
| 运营（非负责） | 仅公开展示信息；**无**底部栏、**无**勾选区 |
| 发布方/代发预览 | `isListingPreview` 时不展示管理区与底栏 |

## 七、需求商机 · 与资源的差异

需求与资源在**认证门槛、发布方隐私、关闭规则、代发规则**上完全一致，仅对接操作名称不同：

| 场景 | 资源详情 | 需求详情 |
| --- | --- | --- |
| 申请方动作（路人） | 底部「申请对接」→ `resource-connect` 勾选己方需求 | 底部「发起匹配」→ 对接表单勾选己方资源 |
| 发布方管理 | 详情页勾选需求批量发起匹配 | 详情页勾选资源批量发起匹配 |

## 八、对接流程权限补充

| 环节 | 谁可操作 |
| --- | --- |
| 提交对接 / 匹配 | 已登录普通用户（非运营）；**登录即可**，不要求名片认证 |
| 对接表单 | 可关联已有资源/需求，或仅填文字说明（`connectUseResource` / `connectUseDemand`） |
| 涉及代发商机的对接 | 先经**运营审批**（代发对接审核 Tab），再进入「待对方确认」 |
| 接收方确认 | `confirmConnectByRecipient`：同意并发起交换名片 |
| 接收方拒绝 | `rejectConnectByRecipient` |
| 申请方取消 | `cancelConnectByApplicant`（仅 applicant 角色；运营账号不能代客户取消） |
| 交换联系方式 | 双方确认（`agreeConnectExchange`） |
| 代发侧确认/交换 | 对接记录中 `getConnectRole = proxyStaff` 时，负责运营可代客户操作 |
| 对接记录中企业名 | 交换前仅脱敏；交换后或己方可见完整 |

运营账号：在**代发详情页底部**替客户勾选发起匹配。**资源/需求详情**结构对称：① **发布方/代发运营**在详情页勾选对方商机批量撮合；② **申请方（路人）**仅见底部主按钮 + 分享，跳转独立页面勾选己方商机后提交（资源详情 → `resource-connect`；需求详情 → 对接表单 `resource_to_demand`）。**不能**在详情页混用两种勾选区。

**全局对接（仅运营）**：运营工作台 → **撮合监管** → **全局对接**，可查看全平台对接记录；列表与详情展示申请方、对接方的企业名称、联系人、手机号及当前状态，**不对双方脱敏**。若对接关联了具体资源或需求，可点击进入商机详情（含已关闭商机）。普通用户无此入口。

**资源对接申请进度**：从资源详情发起的对接，申请记录页仅可「取消对接」，不可「继续申请/重新申请」；取消后须回资源详情重新申请。

**对接进度展示（以当前用户视角）**：

| 标签 | 含义 |
| --- | --- |
| **待我方处理** | 需要您确认对接、交换名片等操作（橙色） |
| **待对方处理** | 已提交，等待对方确认或交换（蓝色） |
| **待平台处理** | 涉及代发商机，等待运营审批（紫色） |

对接列表、对接详情、关联对接均显示上述标签及一句说明，不再仅用「待对方确认」等系统状态。

**完结口径（列表与详情一致）**：双方已交换联系方式（`disclosedContacts` 或状态「已交换名片」）后，不再显示「待我方/待对方/待平台处理」，统一展示「已交换名片」。

**重复申请**：同一商机上若已有未关闭的对接（含进行中、已交换），不可再次发起；资源↔需求成对匹配时，任一方已有进行中对接亦拦截。关闭（「已关闭」「已流失」）后方可重新申请。

**运营两类待办（勿混淆）**：

| 类型 | 含义 | 入口 |
| --- | --- | --- |
| **代发对接审核** | 第三方申请对接代发商机，需运营批准 | 待审中心 → 代发对接 |
| **对接待办** | 代发商机对接流程中，需运营替客户确认/交换 | 我的 → 对接待办 |

**运营内联审核**：`manager` / `admin` 可在「我的」待审列表直接通过/驳回，亦可进入 `admin-review` 详情页操作。

## 九、「我的」页权限展示顺序

> 以下为 `permissionCatalog` 展示顺序；**实际操作门禁**见 §3.1（对接/匹配登录即可，发布需名片）。

普通用户按解锁顺序展示（已开通 / 待解锁）：

1. 浏览需求池与详情  
2. 浏览资源池与详情  
3. 名片认证  
4. 提交需求  
5. 发布资源  
6. 申请对接 / 发起匹配  
7. 营业执照认证  
8. 查看资源附件  
9. 优先推荐  

访客仅展示 1、2 两项。

## 十、平台治理（仅 `staffRole = admin`）

| 操作 | 谁可操作 | 效果 |
| --- | --- | --- |
| 强制下架资源/需求 | 平台管理员 | 商机 `verification` 变为「已关闭」，移出公开展示池；关联未完结对接自动关闭 |
| 禁用普通用户账号 | 平台管理员 | `users.accountStatus = disabled`；用户无法登录与发布/对接；其全部商机自动下架 |
| 解除禁用 | 平台管理员 | 恢复 `accountStatus = active`，用户可重新登录 |

**不可操作**：运营专员（`manager`）不能禁用账号或强制下架他人已发布商机（待审驳回仍走审核流程）。**不能禁用**运营账号。

入口：运营工作台 → **平台治理**；或在商机详情页使用「平台下架」（仅管理员可见）。

| 发布门禁 | `canSubmitListing()` / `ensureSubmitListingAccess()` / `certGate.js` |
| 关闭商机 | `canCloseListing(listingId)` |
| 收藏 | `permissions.canFavoriteContent()` → `toggleFavoriteListing()` |
| 分享菜单 | `permissions.canShareContent()` → `canShareListingContent()` |
| 详情邀请分享 | `permissions.canShareConnectInviteOnDetail()`（访客可用） |
| 资源附件 | `permissions.canViewResourceAttachmentsForListing()` |
| 需求附件 | `permissions.canViewDemandAttachmentsForListing()`（不公开展示） |
| 发布方卡片 | `permissions.canShowPublisherBlockOnDetail()` |
| 发布方完整联系方式 | `permissions.canShowPublisherFullContact()` |
| 池列表隐藏自己 | `shouldExcludeListingFromPublicPool` → `isListingPublisher` |
| 对接角色 | `getConnectRole(submission, phone)`（applicant / recipient / proxyStaff） |
| 账号禁用 | `isAccountDisabled()` / `users.accountStatus` |
| 平台管理员监管视图 | `platformAdminOversight`（仅 `staffRole = admin`） |

**禁止混用**：`isOwnListing` 仅表示「当前用户是否以主人/运营身份操作该商机」（含运营代发管理），**不能**用于关闭、池隐藏、发布方隐私等须严格区分发布方的场景。

**自测**：在项目目录执行 `node miniprogram-mvp/scripts/audit-permissions.js`，应输出「全部权限审计通过」。
