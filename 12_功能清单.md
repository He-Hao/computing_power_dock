## 一、产品定位与边界

### 1.1 产品概述

| 项 | 内容 |
| --- | --- |
| 产品名称 | 算力码头 |
| 载体 | 微信小程序（`miniprogram-mvp`） |
| 目标行业 | 算力、IDC、机房建设、数据中心运营、渠道代理 |
| 商业模式 | B2B 供需撮合平台（MVP 验证期） |
| 核心价值主张 | 真实资源、真实需求、结构化发布、企业认证、智能匹配、人工撮合、对接与名片交换 |

### 1.2 平台做什么 / 不做什么

| ✅ 平台提供 | ❌ 平台不提供 |
| --- | --- |
| 结构化资源/需求发布与公开展示 | 在线支付与资金托管 |
| 企业名片/执照两级认证 | 自动算力调度与交付 |
| 平台初审与运营人工审核 | 交易担保与真实性背书 |
| 智能匹配推荐与批量勾选撮合 | 完整 CRM / 合同管理 |
| 对接申请、确认、名片交换流程 | 企业微信深度集成（规划中） |
| 运营代发、代撮合、全局监管 | 跟进状态（已约谈/已报价/已成交）完整闭环 |

### 1.3 数据运行模式

| 模式 | 配置 | 说明 |
| --- | --- | --- |
| 本地演示 | `useCloud: false` | 数据存于手机本地 Storage，单设备演示；对接/收藏**不跨设备** |
| 云开发多人 | `useCloud: true` + `cloudEnvId` | 多用户共享公池，对接跨设备完成；部署见 [09_云开发部署指南.md](09_云开发部署指南.md) |

**双模式能力差异**：本地模式适合单设备演示；跨用户对接、收藏云端同步、运营审核/代发、账号禁用等**需云端模式**。

### 1.4 设计原则（权限与产品）

1. **身份、认证、关系三层分离**：账号身份 / 认证等级 / 与单条商机的关系独立判定  
2. **一个账号一种身份**：有 `staffRole` 即为运营账号，不可与个人用户身份切换  
3. **发布方 ≠ 代发运营**：关闭、分享等主人操作归发布方或**负责代发运营**（`canStaffManageProxyListing`）；运营不以个人身份对接  
4. **信息分级展示**：公开展示 → 脱敏企业名 → 完整联系方式 → 资源附件逐级放开  
5. **对接方向固定**：需求方→资源方、资源方→需求方，运营不以个人身份操作  

> 完整权限口径见 [10_权限规则.md](10_权限规则.md)（当前 v1.13）。

## 二、用户角色与权限体系

### 2.1 账号身份矩阵

| 身份 | 判定条件 | 可用功能 | 不可用功能 |
| --- | --- | --- | --- |
| **访客** | 未登录 | 浏览公开展示池与详情（不含附件、发布方联系方式） | 收藏、发布、对接、认证 |
| **普通用户** | 已登录，无 `staffRole` | 注册资料、认证、发布、对接、收藏、提交记录 | 运营审核、代发、平台治理 |
| **运营专员** | `staffRole = manager` | 待审中心、代发管理、代撮合、全局对接监管 | 以个人身份发布/对接/认证；在公池以个人对接 |
| **平台管理员** | `staffRole = admin` | 专员全部能力 + 强制下架、禁用/解禁普通用户 | 禁用运营账号；专员不能强制下架 |

**运营账号开通方式**：云开发控制台 → `users` 集合 → 为目标用户添加 `staffRole` 字段（`manager` 或 `admin`）→ 重新登录生效。

### 2.2 认证等级与能力解锁

| 等级 | 条件 | 解锁能力 |
| --- | --- | --- |
| 已注册 | 手机号 + 密码登录 | 浏览、收藏、维护企业资料、**申请对接 / 发起匹配**、人工撮合 |
| 名片认证 | 名片审核通过 | **提交需求、发布资源**（含整机/机房） |
| 执照认证 | 营业执照审核通过 | 名片全部能力 + 查看他人**资源附件** + 执照信任标签 + 列表优先推荐 |

> **对接/发起匹配**：已登录即可（`ensureConnectAccess`），不要求名片认证；详见 [10_权限规则.md](10_权限规则.md) §3.1。

认证为**叠加关系**：有执照视为同时具备名片能力。

### 2.3 与单条商机的关系（动态）

| 关系 | 判定 | 信息/操作差异 |
| --- | --- | --- |
| 路人 | 无下列关系 | 仅公开展示字段 |
| 发布方 | 手机号/openid 与商机归属一致 | 完整发布方信息、管理、关闭、批量匹配 |
| 已授权对接方 | 存在有效对接且状态 ≠ 待平台审核 | 脱敏企业名；交换后完整联系方式 |
| 代发运营 | `publishedByStaff` 且当前运营为负责专员 | 运营视图：客户信息、撮合、对接记录 |

**有效对接**：记录未关闭、未流失。  
**公开展示池隐藏规则**：发布方在资源池/需求池列表中**看不到自己发布的**常规商机（代发已审核商机对他人正常展示）。

### 2.4 权限能力速查

> **表 A** 按认证等级（普通用户）；**表 B** 按与单条商机的关系或身份。操作门禁以 [10_权限规则.md](10_权限规则.md) §3.1 为准。

**表 A · 认证等级（普通用户）**

| --- | --- | --- | --- | --- |

| 收藏 | 引导登录 | ✅ | ✅ | ✅ |
| 提交需求/发布资源 | 引导登录 | 引导认证 | ✅ | ✅ |
| 申请对接（需求→资源） | 引导登录 | ✅ | ✅ | ✅ |
| 发起匹配（资源→需求） | 引导登录 | ✅ | ✅ | ✅ |
| 查看他人资源附件 | ❌ | ❌ | ❌ | ✅ |

**表 B · 商机关系 / 身份**

| 能力 | 发布方 | 已授权对接方（+执照） | 代发运营（负责） | 运营（路人） | admin |
| --- | --- | --- | --- | --- | --- |
| 申请对接自己的商机 | ❌ | — | ❌ | ❌ 走代发 | — |
| 查看己方资源附件 | ✅ | 同路人+执照 | ✅ | — | — |

| 关闭商机 | ✅ | ❌ | ✅ 仅负责代发 | ❌ | 强制下架 |
| 分享菜单 | ✅ | ✅ | ✅ 代发视图 | ✅ | ✅ |
| 详情邀请好友对接 | ❌ | ✅ | ❌ | ❌ | ❌ |
| 平台治理 | ❌ | ❌ | ❌ | ❌ | ✅ |

### 2.5 「我的」页权限展示顺序（普通用户）

按解锁顺序展示已开通 / 待解锁：

1. 浏览需求池与详情  
2. 浏览资源池与详情  
3. 名片认证  
4. 提交需求  
5. 发布资源  
6. 申请对接 / 发起匹配  
7. 营业执照认证  
8. 查看资源附件  
9. 优先推荐  

访客仅展示 1、2 两项。

> **UI 与操作口径**：「我的」页能力标签将「申请对接 / 发起匹配」归入名片档展示；**实际操作**已登录即可（`certGate` / `ensureConnectAccess`），以 [10_权限规则.md](10_权限规则.md) §3.1 为准。

### 3.3 详情页统一结构（资源 / 需求对称）

```
整体说明（完整摘要，优先从 details / 提交记录读取）
  ↓
结构化补充信息（规格、亮点、标签、认证标签）
  ↓
智能匹配推荐区（路人可见推荐，操作分权限）
  ↓
【发布方 / 代发运营】商机管理
  ├── 对接记录
  ├── 匹配需求 / 匹配资源（勾选批量撮合）
  └── 分享 / 关闭
  ↓
【路人 / 申请方】底部主操作
  ├── 资源详情 → 申请对接
  └── 需求详情 → 发起匹配
```

**详情页两种勾选区不可混用**：发布方/代发运营在详情内勾选；申请方跳转独立页勾选己方商机后提交。

## 四、业务品类与编号体系

### 4.1 商机类型（MECE · 8 类）

资源池与需求池**共用同一套四字品类名**（只描述品类，不含销售/采购等交易动词）：

| 类型 | 资源侧说明 | 需求侧说明 |
| --- | --- | --- |
| 算力整机 | 完整服务器/算力设备；买断/交付 | 同左 |
| 硬件配件 | GPU 模组、电源、线缆、存储等 | 同左 |
| 算力租赁 | GPU、机柜、带宽等按租使用 | 含训练、推理、托管场景 |
| 运维维保 | 巡检、备件、驻场维修 | 设备级维护 |
| 机房建设 | 设计、施工、液冷工程 | 一次性建设 |
| 托管运营 | 整站托管、代运营 | 整站托管、渠道合作 |
| 资金支持 | 融资、分期等金融合作 | 项目金融支持 |
| 综合资源 | 以上未覆盖（描述须写清） | 以上未覆盖 |

- 发布/筛选时展示 `resourceTypeHints` / `demandTypeHints`  
- 旧名称（整机、GPU算力、训练算力、IDC托管等）经 `legacy*AliasMap` 自动映射

## 五、功能模块详述

### 5.1 首页（`home`）

| 功能项 | 说明 |
| --- | --- |
| Hero 区 | 品牌标识、平台定位语、双段业务说明 |
| 池入口卡片 | 资源池/需求池在架数量，点击跳转对应 Tab |
| 使用三步 | 「怎么用」卡片：浏览 → 对接 → 交换；链至 `process` 完整流程 |
| 重要提醒 | 固定免责声明（不参与交易、不代收代付） |
| 对接待办 | 登录用户横幅，跳转记录或运营待办 |
| 下一步引导 | 按进度推荐动作（认证/发布/浏览等），可 dismiss（`dismissHomeGuide`） |
| 分享 | 好友分享 + **分享到朋友圈**（`onShareAppMessage` / `onShareTimeline`） |
| 分享落地 | `from=share` 欢迎提示 |
| 同步策略 | 缓存优先 + 15s 节流；普通用户 `refreshPoolPagesFromCloud`，运营 `refreshStaffLaunchFromCloud` |
| 下拉刷新 | 强制拉云端 |

> **说明**：首页当前**无**商机 feed 预览行、**无**发布 FAB；发布入口在资源池/需求池底部 `+` 按钮，或通过下一步引导跳转。

### 5.2 资源池 / 需求池

#### 5.2.1 列表展示

- 脱敏公开展示；**不含**发布方企业名（防列表推断）  
- 默认**不展示本人常规发布**；代发已审核商机对他人正常展示  
- 卡片：类型、地区、规模、价格/预算、匹配分、认证标签、平台代发标识  
- 列表行内标记：资源池「已申请对接」；需求池「已申请匹配」  
- 浏览状态筛选：资源池/需求池均支持（`markResourceViewed` / `markDemandViewed`）  
- 快捷筛选行：我的收藏、执照认证、24 小时内、7 天内（池页顶部 chips）  
- 前端分页 + 云端公池续拉（`loadMorePublicListings`）；`list-pagination-foot` 分页脚  
- 池页分享：好友 + 朋友圈（整池分享）  
- 实现工厂：`utils/poolPageFactory.js` + `utils/poolListPage.js`  

#### 5.2.2 筛选与排序（完整选项）

| 维度 | 选项 | 资源池 | 需求池 |
| --- | --- | --- | --- |
| 关键词 | 标题、类型、地区、编号 | ✅ | ✅ |
| 类型 | 8 类 + 全部 | ✅ | ✅ |
| 地域 | 9 大区 + 全部 | ✅ | ✅ |
| 认证 | 全部 / 执照认证 | ✅ | ✅ |
| 时间 | 全部 / 24h / 7d / 30d | ✅ | ✅ |
| 交期 | 全部 / 现货 / 期货 | ✅ | ✅（抽屉内，整机类资源为主） |
| 收藏 | 全部 / 我的收藏 | ✅ | ✅ |
| 浏览状态 | 全部 / 未浏览 / 已查看 / 已申请 | ✅ | ✅ |
| 排序 | 完整度优先 / 最新发布 | ✅ | ✅ |

筛选抽屉：`filterDrawerOpen`；活跃筛选项计数 `filterActiveCount`。  
跨页筛选意图：`app.globalData.filterIntent`（从首页栏目跳入时带类型/收藏）。

#### 5.2.3 列表操作

| 操作 | 条件 | 行为 |
| --- | --- | --- |
| 查看详情 | 全员 | 进入 `detail` |
| 申请对接 | 资源池卡片底部 | 不进详情，直接走对接表单（防误触） |
| 发起匹配 | 需求池卡片底部 | 不进详情，直接走 `resource_to_demand` 对接表单 |
| 已申请 ✓ | 池列表 | 跳转已有对接记录（`viewConnectRecord`） |
| 收藏/取消 | 已登录 | `toggleFavorite`，云端同步 |
| 发布 FAB | 资源池/需求池底部 `+` | 跳转 `submit?type=resource|demand`；运营拦截走代发 |
| 筛选抽屉 | 两池 | 更多筛选：时间、交期、浏览状态、地域；支持重置 |

运营浏览公池：卡片底部**仅「查看详情」**，不可个人对接/匹配。

#### 5.3.1 信息展示

| 区块 | 可见性 |
| --- | --- |
| 整体说明 | 全员；完整展示不截断 |
| 结构化规格/亮点/标签 | 全员 |
| 核验/认证标签 | 全员（执照/名片/平台代发） |
| 匹配亮点与匹配百分比 | 全员 |

| 代发客户信息卡 | 仅负责运营 + 非分享落地 |
| 资源附件 | 发布方 / 代发运营；路人及已授权对接方需**执照认证**（对接方与路人同口径） |
| 需求附件 | 仅发布方、运营审核（不公开展示） |

**发布方信息分级（资源/需求对称）**：

| 查看者 | 企业名称 | 联系人 / 手机 |
| --- | --- | --- |
| 访客 / 已登录路人 | ❌ | ❌ |
| 发布方本人 | ✅ 完整 | ✅ 完整 |
| 平台管理员（`admin`，非分享落地） | ✅ 完整（云端拉取） | ✅ 完整 |
| 运营专员（`manager`）浏览普通用户发布 | ❌ | ❌ |
| 负责代发运营（非分享落地） | ✅ 完整（**客户信息卡**） | ✅ 完整 |
| 已授权对接方 | ✅ 脱敏企业名 | ❌ 交换前不展示 |
| 交换名片后 | ✅ 脱敏企业名 | ✅ 完整 |

云存储附件经 `resolveCloudFileUrls` 云函数解析（运营审核认证图同理）。

#### 5.3.2 智能匹配区

#### 5.3.3 商机管理（发布方 / 代发运营）

- 对接记录列表（含用户视角进度标签）  
- 匹配需求 / 匹配资源勾选区  
- 分享给好友 + 朋友圈（`from=share` 对外隐藏敏感信息；发布方/代发用分享菜单，路人可用对接邀请按钮）
- 关闭商机（发布方本人，或负责代发运营关闭代发商机；云端级联关闭未完结对接）  
- 平台管理员：详情页「平台下架」（`adminTakeDownListingAsync`）  

#### 5.3.4 申请方操作（路人）

| 商机类型 | 底部按钮 | 跳转 |
| --- | --- | --- |
| 资源 | 申请对接 | `resource-connect` 或 `submit?type=connect` |
| 需求 | 发起匹配 | `submit?type=connect&direction=resource_to_demand` |

### 5.4 提交与发布（`submit`）

#### 5.4.1 表单类型全表

| type 参数 | mode | 页面标题 | 提交按钮 | 门禁 |
| --- | --- | --- | --- | --- |
| `demand` | — | 提交需求 | 提交需求 | 登录 + 名片认证 |
| `resource` | — | 发布资源 | 提交资源审核 | 登录 + 名片认证 |
| `server` | — | 找整机 | 提交整机需求 | 登录 + 名片认证 |
| `room` | — | 找机房/建机房 | 提交项目需求 | 登录 + 名片认证 |
| `match` | — | 申请人工撮合 | 申请人工撮合 | 登录 |
| `connect` | — | 申请对接 / 我有匹配资源 | 提交对接/匹配申请 | 登录即可（非运营） |
| `demand` | `proxy` | 代发需求 | 提交代发需求 | 运营账号 |
| `resource` | `proxy` | 代发资源 | 提交代发资源 | 运营账号 |
| `connect` | `proxy` | 代发资源对接需求 | 提交对接申请 | 运营账号 |

**对接方向**：

- `demand_to_resource`（默认）：需求方在资源详情申请对接  
- `resource_to_demand`：资源方在需求详情勾选自有资源匹配  

#### 5.4.2 整机结构化字段（`type=server`）

产品、规格、数量、报价、交期（**须选择现货/期货** + 明细）、预算等 10 项结构化填写；细节选填。

#### 5.4.3 通用表单能力

| 能力 | 说明 |
| --- | --- |
| 企业资料预填 | 从 `getUserProfile()` 带入联系人、地区、角色等 |
| 附件上传 | 最多 5 个（`MAX_SUBMISSION_ATTACHMENTS`），支持预览/删除 |
| 对接双模式 | 关联已有资源/需求（`connectUseResource` / `connectUseDemand`）或仅填文字说明 |
| 字段校验 | 必填项、交期格式、代发交期必填规则 |
| 自动审核 | `autoApproveListing: true` 时提交后直接进入公开展示池 |
| 提交导航 | 对接成功后统一进入对接记录页 |
| redirect 回跳 | 带 `redirect` 参数完成后返回来源页 |

#### 5.4.4 代发表单特殊规则

- 客户企业/联系人/手机号独立录入；**手机号选填**（不填则客户无法登录查看）  
- 整机：**交期必填**（现货/期货）；地点与企业所在地选填后置  
- 非整机：代发模式下交期（交付/项目周期）亦必填  
- 代发成功后默认进入该条公示详情  

### 5.5 资源对接（`resource-connect`）

- 入口：资源详情、资源池列表「申请对接」  
- 勾选已发布需求或填写对接说明  
- 可选关联己方已发布需求  
- 提交后创建 `UCON-` 对接记录，进入状态机  

### 5.6 账号体系

#### 5.6.1 注册

| 项 | 说明 |
| --- | --- |
| 个人主体（默认） | 手动填写 11 位手机号（`useWechatPhoneAuth: false`） |
| 企业主体（可选） | 微信 `getPhoneNumber` + 部署 `cloudfunctions/getPhoneNumber` |
| 协议与申明 | 注册前须逐项阅读并同意《用户服务协议》《隐私政策》《免责申明》（`legalContent.js`）；每项滚动至底部 + 6 秒倒计时 + 勾选确认 |
| 文档职责 | 协议=完整法律条款；隐私=个人信息处理；免责申明=注册前精简风险告知（不重复协议条文，完整规则以协议为准） |
| 密码 | 强度提示；确认密码校验 |
| 微信绑定策略 | `resolveAuthGateFromCloud`：已绑定手机号的 openid **不可再注册**（`allowRegister: false`，注册 Tab 禁用）；已绑定号登录时手机号锁定（`phoneLocked`） |
| 多账号 | 同一微信可注册多个**不同**手机号；云端以**手机号**隔离数据 |
| 账号禁用 | `accountStatus=disabled` 时登录/会话校验拦截（`assertActiveUserAccount`） |
| 注册后去向 | ① 带 redirect 回跳 ② 无 redirect 进 onboard ③ 分享回流跳过 onboard 直达详情 |
| 退出登录 | `logoutUser()` 清空本机缓存；**当前 UI 无退出按钮**（会话失效时自动触发；切换账号需清缓存或换设备） |

#### 5.6.2 登录

- 手机号 + 密码  
- 设备会话校验（`validateDeviceSession`）  
- openid 绑定（`ensureOpenidBound`）  
- 分享意图恢复（`shareIntentKey`）  
- 运营入口：`login?mode=admin`  
- 退出登录清空本机缓存  

#### 5.6.3 欢迎引导（`onboard`）

| 选项 | 后续动作 |
| --- | --- |
| 找算力 | 跳转需求相关入口 |
| 供算力 | 跳转资源发布 |
| 先逛逛 | 进入首页 |

已登录用户修改密码，云端 `changePassword` API。

### 5.7 企业认证（`certify`）

#### 5.7.1 认证等级

| 等级 | 提交材料 | 审核后 |
| --- | --- | --- |
| 名片认证（card） | 个人名片 + 企业基础信息 | 解锁提交需求 / 发布资源（对接/匹配仅需登录，见 `certGate.js`） |
| 执照认证（license） | 营业执照 | 解锁资源附件查看 + 信任标签 + 优先推荐 |

#### 5.7.2 认证流程能力

- 升级模式：名片已通过 → 执照升级（名片字段锁定）  
- 只读模式：审核中/已通过时部分字段不可编辑  
- 认证记录：`cert-records` / `cert-record`  
- 分享回流：认证页记住 `redirect`，通过后回跳并提示「继续申请对接」  
- 运营账号不可提交认证  

#### 5.7.3 认证状态文案

| 状态 | 说明 |
| --- | --- |
| 待审核 | 平台已收到，等待撮合经理初审 |
| 认证中 | 1–3 个工作日完成审核 |
| 已认证 | 展示认证标签，参与优先推荐 |

### 5.9 提交记录（`records` / `record`）

#### 5.9.1 记录类型

| type | 显示名 |

#### 5.9.2 列表能力

- 类型筛选：全部 / 对接 / 资源 / 需求（**不含** `certify`；认证独立页 `cert-records`）  
- **二级状态筛选**（随类型变化）：  
  - 对接：全部 / 待我处理 / 进行中 / 待平台 / 待对方 / 交换中 / 已完成 / 已关闭  
  - 资源/需求：全部 / 待审核 / 已发布 / 已关闭 / 已驳回  
- 关键词搜索  
- 分页（`pagedList`）  
- 列表内关闭商机（发布方或负责代发运营，`closeListing`）  
- 对接记录：用户视角进度标签 + 迷你时间线

#### 5.9.3 记录详情操作

| 角色/状态 | 可用操作 |
| --- | --- |
| 对接接收方 | 同意并发起交换名片（`confirmConnectByRecipient`）/ 拒绝（`rejectConnectByRecipient`）/ 确认交换 |
| 对接申请方 | 确认交换 / 取消对接（`cancelConnectByApplicant`） |
| 发布方 | 关闭商机 |
| 负责代发运营 | 关闭负责代发的商机 |
| 运营 | 代发对接确认/交换；审批入口 |
| 交换完成后 | 查看双方完整联系方式（`disclosedContacts`）+ **线下合作风险提示**（`getConnectSuccessRiskNotice`） |
| 关联对接 | 同商机其他对接记录列表（`linkedConnects`） |
| 附件 | 预览；待审发布类记录在允许时可编辑附件（`canEditSubmissionAttachments`） |
| 公示预览 | 非 connect 类记录可跳转关联商机详情 |

**资源对接进度限制**：从资源详情发起的对接，记录页**仅可取消**，不可「继续申请」；取消后须回详情重新申请。

### 5.11 运营 · 审核

#### 5.11.1 待审队列分类

| Tab key | 内容 |
| --- | --- |
| `all` | 公示 + 全部提交类待审 |
| `listing` | 资源/需求公示初审 |
| `certify` | 企业认证 |
| `business` | 人工撮合等商机类（不含认证、代发对接） |
| `proxyConnect` | 第三方申请对接代发商机 |

#### 5.11.2 审核操作

- 公示：通过 → 进入公开展示池；驳回 → 保持待审/关闭  
- 认证：通过 → 更新用户 `certLevel`；驳回 → 可重新提交  
- 代发对接：批准 → 进入对方确认；驳回 → 关闭  
- 认证图片：`resolveCloudFileUrls` 读取云存储（客户端无法直接读他人文件）

### 5.17 撮合流程说明（`process`）

5 步可视化（门禁与 [10_权限规则.md](10_权限规则.md) §3.1 一致）：

1. **浏览与对接** — 浏览公开展示池；**已登录即可**申请对接/发起匹配  
2. **发布与认证** — **发布**需求/资源需名片认证；执照认证解锁附件与优先推荐  
3. **对方确认** — 接收方同意对接并发起交换名片  
4. **信息公示** — 双方同意交换后查看联系方式  
5. **跟进沉淀** — 记录报价、约谈、成交或流失（状态文案已预留，运营闭环待完善）

## 六、核心业务流程

### 6.1 访客浏览流程

```
打开小程序 → 首页/资源池/需求池 Tab
  → 浏览列表（筛选/排序/搜索）
  → 进入详情（完整公开展示信息）
  → 点击发布/对接/收藏 → login-gate 或 login 引导
```

### 6.2 新用户注册与引导

```
功能入口触发注册（带 redirect）
  → login 注册 Tab → 逐项同意协议三件套 → 手机号+密码
  → 注册成功
      ├─ 有 redirect → 回来源页
      ├─ 分享回流 → 跳过 onboard → 目标详情
      └─ 主动注册 → onboard（找算力/供算力/先逛逛）→ 对应入口
```

### 6.3 名片认证 → 发布 → 公开展示

```
我的 → 企业认证 → 上传名片+企业信息 → 提交
  → 运营待审中心 → 通过
  → 用户获得发布能力（对接/匹配已登录即可，无需名片）
  → 提交需求/资源 → [autoApprove? 自动 : 人工] 审核
  → 进入公开展示池（URES-/UDEM-）
  → 系统智能匹配推荐互补商机
```

### 6.4 需求方申请对接资源（demand_to_resource）

```
资源池/详情 → 申请对接
  → [登录即可，不要求名片认证]
  → resource-connect：勾选己方需求或填说明
  → 创建 UCON- 记录
  → [涉代发? 待平台审核 → 运营批准]
  → 待对方确认（资源方）
  → 资源方「同意并发起交换名片」
  → 申请方确认交换
  → 已交换名片（双方可见联系方式）
```

### 6.5 资源方匹配需求（resource_to_demand）

```
需求详情 → 发起匹配
  → 勾选己方已发布资源 + 说明
  → 创建 UCON- 记录
  → 后续流程同 6.4（角色对调）
```

### 6.6 发布方批量匹配

```
自己的资源/需求详情 → 商机管理
  → 智能推荐/搜索 → 勾选目标商机（最多 20）
  → 批量提交对接
  → 每条创建独立 UCON- 记录
```

### 6.7 运营代发全流程

```
运营登录 → 我的（运营工作台）
  → 代发管理 → 代发资源/需求
  → 填写客户资料 → 提交 → 待审 → 通过 → 公开展示池
  → 详情查看匹配推荐 → 勾选替客户对接
      ├─ 双边代发 → 自动完结
      ├─ 单边代发 → 对方确认+交换流程
      └─ 第三方申请对接代发商机 → 代发对接审核 → 批准 → 确认流程
```

### 6.8 关闭商机

```
发布方 → 详情/记录 → 关闭此商机
  → 商机 verification=已关闭，移出公开展示池
  → 云端级联关闭所有未完结关联对接
  → admin 强制下架同理（附加账号禁用时全量下架）
```

### 6.9 好友分享回流

```
分享链接（from=share + 商机 ID）
  → 打开小程序，写入 shareIntent
  → 访客浏览详情（可转发「邀请好友对接」，不可用分享菜单）
  → 点击对接/收藏 → 登录/注册
  → 完成后回目标详情
  → 申请对接（登录即可，不要求名片）→ 进入 6.4 流程
  → [若来源为发布且未名片认证] 认证页记 redirect → 通过后回跳发布
```

### 6.10 人工撮合申请（`type=match`）

```
已登录用户 → submit?type=match
  → 填写撮合说明与意向
  → 创建 SMAT- 提交记录
  → 运营待审中心（business Tab）→ 通过/驳回
  → 运营线下跟进（无完整跟进状态 UI，见 §12.1）
```

### 6.11 认证驳回与重提

```
certify 提交 → 运营驳回
  → cert-records 可见驳回原因
  → 用户修改材料重新提交（执照升级时名片字段锁定）
  → 通过后更新 certLevel，解锁对应能力
```

### 6.12 账号禁用与解禁（admin）

```
admin → 平台治理 → 手机号查找用户
  → 禁用：accountStatus=disabled → 无法登录；商机全部下架
  → 解禁：恢复 active → 用户可重新登录（商机不自动恢复，须重新发布）
```

## 七、对接状态机与规则

### 7.1 阶段定义（`utils/connectStage.js`）

| 阶段常量 | 含义 |
| --- | --- |
| `PLATFORM_REVIEW` | 待平台审核（涉代发第三方对接） |
| `WAIT_RECIPIENT_CONFIRM` | 待接收方确认对接 |
| `WAIT_APPLICANT_EXCHANGE` | 待申请方确认交换 |
| `WAIT_RECIPIENT_EXCHANGE` | 待接收方确认交换 |
| `EXCHANGED` | 已交换名片 |
| `CLOSED` | 已关闭/已流失 |

### 7.2 标准流转

```
已申请对接
  → [可选] 待平台审核 → 运营批准
  → 待对方确认
  → 接收方「同意并发起交换名片」（进入 `WAIT_APPLICANT_EXCHANGE` 或 `WAIT_RECIPIENT_EXCHANGE`）
  → 待交换确认（申请方 `agreeConnectExchange` 确认）
  → 已交换名片
```

### 7.3 用户视角进度标签

| 标签 | 颜色语义 | 含义 |
| --- | --- | --- |
| 待我方处理 | 橙 | 需确认对接或交换名片 |
| 待对方处理 | 蓝 | 已提交，等对方操作 |
| 待平台处理 | 紫 | 涉代发，等运营审批 |
| 已交换名片 | — | 完结，不再显示待办 |

### 7.4 业务规则

| 规则 | 说明 |
| --- | --- |
| 重复申请拦截 | 同一商机对存在**未关闭**对接时不可再发起（**含已交换**，`isActiveConnectSubmission` 仅排除已关闭/已流失） |
| 成对拦截 | 资源↔需求成对存在进行中对接亦拦截 |
| 关闭后可重申 | 状态为已关闭/已流失后可重新申请 |
| 云端状态校验 | `patchSubmission` 校验合法状态转换 |
| 完结口径 | `disclosedContacts` 或状态「已交换名片」后统一完结展示 |
| 订阅授权 | 提交/确认/交换时请求订阅消息（可配置跳过） |

### 7.5 代发对接特殊规则

| 场景 | 行为 |
| --- | --- |
| 运营申请 → 普通用户 | 用户收到对接待办 |
| 普通用户 → 运营代发 | 运营账户收到待确认/交换 |
| 运营代客户勾选（双边代发） | 直接自动完结，不推送确认 |
| 第三方对接 + 双边代发 | 平台审核通过后自动完结 |
| 运营代客户勾选（单边） | 仍走对方确认与交换 |
| 运营代客户勾选 | 无需代发对接再审 |
| 第三方申请代发商机 | 待审中心 → 代发对接审核 |
