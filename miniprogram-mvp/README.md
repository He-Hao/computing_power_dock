# 算力码头小程序 MVP

这是平台的最小产品单元，用于验证用户是否愿意因为真实资源、真实需求和人工撮合而留下信息。

## 1. 当前功能

### 首页

- 紧凑 Hero + 单卡片主内容（怎么用 / 发布 / 商机速览）
- 商机预览为精简 feed 行，点进详情；无跳转需求池/资源池入口

### 资源池 / 需求池

- 脱敏资源池和需求池展示
- **默认不展示本人发布的**资源/需求（可在「我的 → 提交记录」查看公示预览）
- 关键词搜索
- 类型筛选 + 地域筛选
- 完整度优先 / 最新发布排序
- 列表页可直接申请对接（不会误触进入详情）

### 商机详情

- 匹配亮点、核验状态、结构化信息
- 智能匹配推荐（根据地域、类型、标签推荐互补商机）
- 申请对接（登录即可）

### 提交与认证

- 提交需求（必填类型、地区、描述；规模/预算等选填默认展开，24 小时内推荐资源方）
- 发布资源（需先注册，审核通过后在公开展示池展示；`config.js` 可设 `autoApproveListing: true` 自动审核）
- 找机房/建机房
- 申请人工撮合
- 申请对接
- **用户注册**（手工确认手机号 + 免责申明；默认预填 11 位号码，可修改后确认）

### 手机号确认（当前默认：手工模式）

### 个人主体小程序（当前默认）

个人主体**无法**使用微信「一键授权手机号」，注册页采用**手动填写**：

1. 勾选免责申明
2. 输入本人 11 位手机号
3. 点击 **同意并注册**

`utils/config.js` 中请保持：

```javascript
useWechatPhoneAuth: false,
defaultManualPhone: ""   // 留空，不要预填测试号
```

### 企业主体 + 微信授权（后续可选）

小程序完成**企业认证**并开通手机号权限后，可切换为微信 `getPhoneNumber`：

1. 在 `utils/config.js` 设置 `useWechatPhoneAuth: true`
2. 开通云开发并部署 `cloudfunctions/getPhoneNumber`
3. 注册页显示「微信授权手机号」按钮
- 企业认证两级：名片认证（上传个人名片）/ 营业执照认证（上传执照），各等级标签与权益不同

### 我的

- 进入页面**先展示本地缓存**，再在后台同步云端数据（15 秒内重复进入不会重复拉取）；运营账号同步待审队列，不再每次刷新整个资源/需求公开展示池
- 顶部展示账号（姓名 + 手机号）
- **一个账号一种角色，不可切换**：
  - **普通用户**（无 `staffRole`）：企业资料、我的商机、对接待办、发布与对接
  - **运营账号**（已开通 `staffRole`）：登录后打开底部「我的」即进入运营工作台（待审、代发管理）；**不能**以个人身份发布或申请对接
- 代发商机的企业/联系人取自**代发录入时填写的客户资料**
- 退出登录后需重新登录；运营权限由云端 `staffRole` 决定

## 2. 登录后操作流程

### 注册成功后的三种去向

1. **从功能入口注册**（带 `redirect` 参数）：注册完成后直接回到原页面（如提交需求、查看详情）
2. **主动注册**（无 redirect）：进入欢迎引导页，选择「找算力 / 供算力 / 先逛逛」
3. **老用户**：正常使用，「我的」页顶部显示下一步待办

### 好友分享资源后的回流

1. 分享链接带 `from=share`，打开后系统记住「要查看/对接哪条资源」
2. **已有账号**：登录拦截页可直接登录，完成后回到资源详情
3. **新用户**：注册后同样回到资源详情（跳过欢迎引导）
4. 未完成名片认证时，认证页会记住回跳地址；提交认证后回到该资源
5. 认证通过后再次进入详情，会提示「继续申请对接」

### 用户权限

完整口径见项目根目录 [10_权限规则.md](../10_权限规则.md)（v1.13），代码实现见 `utils/permissions.js`、`utils/certGate.js`。

| 等级 | 条件 | 权限 |
| --- | --- | --- |
| 访客 | 未登录 | 浏览需求池与资源池（含详情，不含附件与发布方）；操作引导登录 |
| 已注册 | 手机号 + 密码登录 | 同上，可管理个人资料、收藏、**申请对接 / 发起匹配** |
| 名片认证 | 上传名片并通过审核 | **提交需求、发布资源** |
| 已认证 | 营业执照认证通过 | 查看他人资源附件 + 信任标签 + 优先推荐 |

同一微信可注册多个不同手机号账号（例如测试 A/B）。退出登录会清空本机缓存；云端以 **手机号** 隔离资料与提交记录。若升级后仍看到旧账号数据，请重新部署 `tradeApi` 云函数并退出后重新登录。

### 访客态与登录拦截（对齐原型 v1.3）

- **访客首页**：展示业务栏目、精选资源与最新需求
- **资源池 / 需求池 Tab**：访客均可浏览列表与详情
- **发布 / 对接 FAB**：跳转登录或 `login-gate` 拦截页
- **详情页（访客）**：可看完整信息；点击「申请对接」「我有匹配资源」等操作时再引导登录
- **我的（访客）**：登录 + 注册双按钮卡片

### 用户注册抓手

当前版本围绕四个动作设计：

- 提交需求，24 小时内推荐 3 家资源方
- 发布资源，进入首批推荐资源池
- 申请对接真实需求（登录即可）

## 4. 当前数据方式

### 本地演示（默认）

`utils/config.js` 中 `useCloud: false`，数据存在手机本地。

### 云开发多人模式

设置 `useCloud: true` 并填写 `cloudEnvId` 后启用。部署步骤见项目根目录 [09_云开发部署指南.md](../09_云开发部署指南.md)。

云端模式下：

- 资源/需求/提交记录存入云数据库
- 多用户共享公池，对接可在不同手机完成
- 页面进入时自动从云端同步

`utils/data.js` 提供的主要方法：

| 方法 | 用途 |
| --- | --- |
| `getResources()` | 获取资源列表（不含本人常规发布；已审核代发资源仍展示） |
| `getDemands()` | 获取需求列表（不含本人常规发布；已审核代发需求仍展示） |
| `getItem(id)` | 获取单条商机 |
| `getRelatedMatches(id)` | 获取智能匹配推荐 |
| `filterItems(items, options)` | 筛选列表 |
| `sortItems(items, sortBy)` | 排序列表 |
| `saveSubmission(payload)` | 保存提交记录 |
| `getSubmissions()` | 获取全部提交记录 |
| `getSubmission(id)` | 获取单条提交记录 |
| `saveUserProfile(profile)` | 保存用户企业资料 |
| `getUserProfile()` | 读取用户企业资料 |

## 7. 运营权限（staffRole）

运营审核不再使用硬编码账号。开通方式：

1. 在云开发控制台 → 数据库 → `users` 集合
2. 找到目标用户记录，添加字段 `staffRole`，值为 `manager`（运营专员）或 `admin`（平台管理员）
3. 用户重新登录后，**运营账号**自动进入运营工作台能力；**普通用户**仅可使用发布与对接功能

### 角色说明

| 角色 | 判定条件 | 可用功能 |
|------|----------|----------|
| 普通用户 | 无 `staffRole` | 发布/对接、企业认证、我的商机 |
| 运营账号 | `staffRole` = manager / admin | 待审中心、代发管理、代发对接审核 |

运营账号**不能**切换为普通用户，也不能在资源池/需求池以个人身份发布或申请对接；浏览公开展示池时卡片底部仅显示「查看详情」，需替客户对接请进入「代发管理」。

### 代发管理（运营替客户录入）

路径：**我的**（运营账号自动展示工作台）→ **代发管理**（`pages/ops-proxy/ops-proxy`）

- 顶部可一键「代发资源 / 代发需求」，跳转 `pages/submit/submit?mode=proxy`
- 列表支持按类型（资源/需求）、状态（待审/已发布）、关键词筛选
- 代发资源详情可勾选需求发起匹配（替客户对接）；代发需求详情可勾选资源发起匹配；**默认展示最接近的 3 条智能推荐**，支持预览，可「查看更多」展开最多 20 条后勾选匹配
- **代发资源对接**：在代发资源详情页直接选择目标需求（推荐/搜索），点击「替客户对接」进入对接表单提交，无需跳转需求池
- **代发对接审核**：第三方申请对接代发商机时，需运营在「待审中心 → 代发对接审核」批准后才进入对方确认流程；运营代客户勾选的匹配无需再审
- **代发对接提醒规则**：
  - 运营申请 → 普通用户接收：用户收到对接待办提醒
  - 普通用户申请 → 运营代发接收：运营账户模式下收到待确认/待交换提醒
  - **运营代客户勾选发起匹配**（代发资源对接代发需求）：双方均为代发时直接自动完结，不推送确认/交换提醒；单边代发仍走对方确认与交换流程
  - 代发需求侧运营在单边代发对接中点击「同意并发起交换名片」时，云端 `patchSubmission` 会识别 `recipientProxyStaffOpenid` / 需求 listing 的 `proxyStaffOpenid`（修改云函数后需重新部署 `tradeApi`）
  - **本人勾选匹配**（用自己的资源匹配自己的需求等）：系统会正确展示「同意并发起交换名片」，不再误显示为仅可取消
  - 第三方申请对接且双方均为代发：平台审核通过后自动完结，不推送确认提醒，但保留对接记录
- 底部固定「代发资源 / 代发需求」按钮
- 每条代发卡片默认展示最接近的 3 条匹配推荐，可预览；点击「查看更多匹配」进入详情页展开完整列表并勾选发起匹配
- 代发成功后默认进入该条公示详情；返回代发管理页可继续处理其他记录
- **代发需求表单**（整机）：交期必填（须注明期货/现货），地点与企业所在地为选填并后置；非整机类型交期（交付/项目周期）在代发模式下亦必填；**客户手机号选填**（不填则暂无法归属客户账号登录查看）
- **代发详情隐私**：负责该条代发的运营从任意非分享入口进入详情均可查看客户信息；分享链接（`from=share`）对外一律隐藏客户信息
- **代发详情稳定性**：公开展示池同步时不再用脱敏数据覆盖本地代发完整记录，避免运营视图闪一下后消失；代发管理在列表为空时会立即拉云端，并合并待审代发记录统计

运营工作台内容直接展示在「我的」页（审核待办 / 代发业务 / 运营看板 / 测试数据）；`platform-ops` 页面仅作旧链接兼容跳转。云端模式下运营账号可在「测试数据」区 **导入/清空演示数据**（`config.js` → `enableDemoSeedTools`）；亦可使用 `scripts/seed-demo-users.md` 中的云函数方式。

**运营审核认证图片**：审核页通过云函数 `resolveCloudFileUrls` 读取用户上传的云存储文件（客户端无法直接读取他人文件）。修改云函数后需重新部署 `tradeApi`；若图片仍无法显示，请确认 `utils/config.js` 的 `cloudEnvId` 与上传时环境一致。

## 8. 订阅消息

在 `utils/config.js` 填写微信订阅消息模板 ID：

```javascript
subscribeTemplates: {
  connectStatus: "你的模板ID",
  connectPending: "你的模板ID"
}
```

用户提交对接、确认对接、交换联系方式时会弹出授权（未配置模板 ID 时自动跳过）。

## 9. 常见问题：Failed to fetch

开发者工具出现 `清除登录状态失败` 或 `webapi_getwxaasyncsecinfo:fail Failed to fetch`，是**工具连不上微信服务器**，不是小程序代码问题。

**快速处理：**

1. 开发者工具 → 设置 → 代理 → **不使用任何代理**
2. 关闭 VPN / Clash / 公司防火墙限制
3. 切换账号：在小程序内 **我的 → 退出登录**，不必依赖 IDE「清除登录状态」
4. 详细排查步骤见项目根目录 [09_云开发部署指南.md](../09_云开发部署指南.md) 第九节

本地诊断脚本（PowerShell）：

```powershell
.\miniprogram-mvp\scripts\check-wechat-dev.ps1
.\miniprogram-mvp\scripts\reset-wechat-devtools.ps1
```

| 运营浏览公开展示池 | ✅ 仅查看详情，不可个人对接 |
| 名片认证 gating | ✅ 对接/发布需认证 |

### 本次评审修复项

1. **代发商机匹配区修复**：运营账号管理自己代发的资源/需求时，详情页「商机管理」内可正常显示「匹配需求/匹配资源」勾选区（此前因运营身份被误拦截）
2. **代发需求可见性修复**：公开展示池同步时保留代发运营归属字段；代发管理可从提交记录回查需求；云端 `listListings` 按 ID 前缀兜底归类需求池
3. **代发商机公池展示修复**：运营账号浏览资源/需求池时，所有「平台代发」商机一律可见（不再因运营身份或 `ownerOpenid` 被误判为发布方而隐藏）；`getResources` / `getDemands` 会合并代发管理中已审核商机
4. **云同步补全**：`syncData` 增加 `applicantProxyStaffOpenid/Phone` 查询，代发资源侧发起的对接可同步到运营账号
5. **对接记录页**：「待平台审批」状态增加「前往审批」按钮，跳转 `admin-review`
6. **云端状态机**：`patchSubmission` 对 connect 类型校验 status 合法转换，防止异常 patch
7. **提交后导航**：需求方申请对接资源成功后，统一进入对接记录页（与 resource_to_demand 一致）
8. **清理废弃代码**：移除未使用的 `proxyConnectResourceId` 全局状态（代发对接改走 URL 参数 `sourceResourceId`）

### 已知限制（非阻塞，后续可优化）

- 记录量极大（单账号 300+ 对接）时，`syncData` 仍可能截断最早记录；可按 `updatedAt` 做增量同步
- 运营账号在 `getConnectRole` 中不返回 applicant/recipient，客户需用本人账号取消对接
- 云端模式需重新部署 `tradeApi` 后上述修复才生效
