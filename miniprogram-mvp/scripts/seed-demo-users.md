## 数据说明

| 项目 | 值 |
| --- | --- |
| 用户数量 | 4 |
| 手机号 | `18800000000` ～ `18800000003`（每个 +1） |
| 登录密码 | `Demo1234`（至少 8 位且含字母与数字） |
| 每人资源 | 1 条「服务器整机」资源（已初审，公开展示） |

## 业务编号前缀（查询用）

| 前缀 | 含义 | 示例 |
| --- | --- | --- |
| `URES-` | 公示资源（资源池） | `URES-20250610-143022-K7M3` |
| `UDEM-` | 公示需求（需求池） | `UDEM-20250610-143055-P2N8` |
| `UCON-` | 对接/匹配记录 | `UCON-20250610-144012-R5T1` |

演示种子数据使用固定后缀：`URES-SEEDR00001`、`SRES-SEEDR-00001` 等。旧版 `SUB-` 前缀仍兼容读取。

## 常见问题：-504003 超时

若出现 `errCode: -504003`，说明云函数执行超时（默认仅 3 秒）。4 条数据通常一次导入完成；若仍超时，请确认 `tradeApi/config.json` 超时设为 60 秒并 **重新部署** `tradeApi` 云函数。

## 清空演示数据

通过云函数 `tradeApi` 的 `clearDemoData` 操作（见下方「方式二」），或在云开发控制台手动删除测试用户与关联记录：

- 删除 `18800000000`～`18800000099` 测试用户（含历史 100 用户批量数据）
- 删除其发布的资源、需求及对接申请记录

> 小程序「我的」页已移除开发工具入口；演示数据仅通过脚本 / 云函数维护。

## 方式一：云函数导入（推荐）

1. 部署云函数 `cloudfunctions/tradeApi`（右键 → **上传并部署：云端安装依赖**，确保 `config.json` 含 `timeout: 60`）
2. 确认 `utils/config.js` 中 `useCloud: true`
3. 在云开发控制台或本地脚本调用 `seedDemoData`（见方式二）
4. 运营工作台底部 **测试数据** → 点击 **导入演示数据**，确认后开始导入
5. 导入完成后，用 `18800000000` / `Demo1234` 登录验证；资源池、需求池应各有 4 条整机数据

如需重置：先 **清空演示数据**，再重新导入。

## 方式二：云函数测试面板

微信开发者工具 → **云开发** → **云函数** → `tradeApi` → **测试**，输入：

```json
{
  "action": "seedDemoData",
  "payload": {
    "username": "manager",
    "password": "2580",
    "startPhone": 18800000000,
    "totalCount": 4,
    "offset": 0,
    "batchSize": 4,
    "userPassword": "Demo1234"
  }
}
```

注意：演示数据导入需在 **已开通 staffRole 的运营账号** 下，从「平台运营」进入后操作。`userPassword` 是批量创建的测试用户登录密码（默认 `Demo1234`，须至少 8 位且含字母与数字）。

清空演示数据（云函数测试）：

```json
{
  "action": "clearDemoData",
  "payload": {
    "username": "manager",
    "password": "2580",
    "startPhone": 18800000000,
    "totalCount": 100,
    "offset": 0,
    "batchSize": 10
  }
}
```

`totalCount: 100` 用于清除 `18800000000`～`18800000099` 整段历史演示手机号。
