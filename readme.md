# 算力贸易撮合平台

这是一个面向算力产业链的商机撮合平台项目，目标是帮助资源方、需求方、IDC、机房建设商和代理商更快找到真实合作对象。

## 小程序 MVP 功能

当前小程序已实现以下能力：

### 首页

首页布局：**上区双白卡**（品牌说明 + 怎么用），浏览商机请通过资源池/需求池进入。

| 区块 | 说明 |
| --- | --- |
| **品牌卡** | 定位 + 两段文字说明；底部资源池/需求池入口卡片（数量 + 池名 + 查看，一体可点） |
| **怎么用** | 三步引导（图标 + 双行说明），可进完整流程 |
| **待办条** | 对接待处理或下一步引导（登录用户，有则显示） |
| **重要提醒** | 页面底部独立免责说明区 |

已移除：首页资源/需求预览列表。

### 资源池 / 需求池

- 关键词搜索
- **列表分页**：首屏展示 20 条，上拉加载更多；筛选/排序后从第一页重新展示；云端公开展示池本地缓存用尽后自动续拉下一页（每页最多 50 条）
- 类型筛选（资源/需求 **共用同一套 8 类**：算力整机 → 硬件配件 → 算力租赁 → 运维维保 → 机房建设 → 托管运营 → 资金支持 → 综合资源；选类型后展示说明；旧名称如训练算力/IDC托管自动映射）
- **硬件配件**发布/需求表单：配件名称、规格型号、数量、报价/预算、交期、地点等结构化字段（不再套用算力租赁的「资源规模/租期」表单）
- 资源池/需求池「更多筛选」支持 **交期类型**：全部 / 现货 / 期货（从标题、交期、标签等自动识别）
- **更多筛选**：企业所在地、发布时间（7天/30天）、浏览状态（未浏览/已查看/已申请）；首屏快捷筛选含执照认证、我的收藏、7 天内
- **收藏**：列表卡片、详情页、对接申请表单均可点星标收藏；「我的」可快速进入收藏列表
- 完整度 / 最新发布排序（完整度按信息填写质量计算，人人相同）
- 列表页右侧「接」小图标可快速跳转对接申请（完整对接仍在详情页底部）
- **浏览标注**：点进详情即标记「已查看」；资源池「已申请」为需求方对接资源，需求池「已申请」为资源方匹配该需求
- **对接返回体验**：从对接页返回详情/列表时不再强制跳转对接记录；已申请过的资源留在原页面，需查看进度时点「✓」或详情底栏「查看申请进度」
- **资源列表标签**：不再展示「资源已初审」；「已查看/已申请」与类型标签同一行展示，避免卡片行高跳动；标签行自动去掉与类型/地区重复项（如「整机」不再出现两次）
- **资源发布方隐私**：发布方企业与联系人默认不展示；**访客与已登录路人**均不可见；平台**管理员**（`staffRole=admin`）、发布者本人、已授权对接方（先脱敏企业名，交换联系方式后可见联系人/手机号）可查看；运营专员浏览普通用户商机时不展示发布方区，代发商机通过「客户信息」卡片查看
- 代发资源/需求审核通过后，与普通用户发布一样进入**资源池/需求池**公开展示（运营账户浏览池时同样可见）

### 商机详情

- 需求/资源详情**统一布局**：顶部卡片与列表池、对接表单共用 `pool-card-v2` + `poolFacts`（📍📐💰）；摘要全文展示；补充规格列表
- 列表卡片 `poolFacts` 与详情顶栏、对接表单目标卡片共用同一套 `buildListingViewLayout` 数据
- 智能匹配推荐（仅发布者本人可见）；详情页对本人/代发运营展示「已对接需求/资源」列表
- 申请对接（登录即可，表单填写企业名称/联系人/手机号）

### 提交与认证

- 提交需求、发布资源、找机房、人工撮合、申请对接
- **用户注册与登录**：填写姓名、手机号、登录密码（至少8位，须同时包含字母与数字）+ 勾选免责申明；登录页使用手机号+密码；「我的」页可修改密码、退出登录
- **需求提交**：必填类型、需求描述；企业所在地、规模、预算等为选填，提交后完整展示
- **结构化发布**：资源/需求按类型展示差异化字段；支持上传附件（配置单、方案书等，最多5个）；需求附件不公开展示，**资源附件需完成营业执照认证后可查看**
- **附件修改**：审核前可在提交记录详情页增删附件并保存
- **发布即审核**：资源/需求提交后先进入平台审核，**审核通过后才在公开展示池展示**；演示环境可在 `config.js` 设 `autoApproveListing: true` 自动完成审核
- 企业认证两级：名片/营业执照；**名片认证**仅需企业名称、联系人、手机号、认证说明与名片照片（不含企业所在地与角色）；**营业执照认证**需填写所在地与角色；通过后仅可查看；升级执照时锁定名片已填项，说明可补充、空白项可完善
- 用户可关闭自己发布的需求或资源（从公开展示池下架）；**负责该条代发商机的运营**也可在代发详情页关闭下架
- 对接申请仅支持需求方对接资源方，可申请时勾选己方需求；**涉及平台代发的对接需先经运营审批**；资源方可从需求详情「我有匹配资源」勾选已发布资源发起匹配；**发布方本人**与**运营代发**均在详情页底部使用同款勾选区批量发起匹配/申请
- 表单自动记住企业基础信息，减少重复填写

### 登录后操作流程（重新规划）

用户完成注册后，按以下路径引导：

| 场景 | 注册成功后去向 |
| --- | --- |
| 从某功能入口注册（有 redirect） | 直接回到原页面继续操作 |
| 主动注册（无 redirect） | 进入「欢迎引导页」，选择目标后跳转 |
| 老用户再次打开 | 首页 / 上次页面，「我的」展示下一步待办 |

**欢迎引导页**（`pages/onboard/onboard`）三选一：

1. **我要找算力** → 提交需求页
2. **我要供算力** → 发布资源页
3. **我先逛逛** → 首页

**用户权限**（完整规则见 [10_权限规则.md](10_权限规则.md)）：

| 等级 | 条件 | 能做什么 |
| --- | --- | --- |
| 访客 | 未登录 | 浏览需求池与资源池（含详情，不含附件与发布方） |
| 已登录 | 手机号 + 密码登录 | 同上，可管理个人资料、收藏 |
| 名片认证 | 上传名片并通过审核 | 提交需求、发布资源、申请对接 / 发起匹配 |
| 已认证 | 营业执照认证通过 | 查看他人资源附件 + 信任标签 + 优先推荐 |

**「我的」页待办引导**会根据当前状态自动提示下一步（完善资料、提交认证、查看审核进度等）。

### 我的

- **一个账号一种角色**：有 `staffRole` 即为运营账号，否则为普通用户，**不可在同一账号内切换**
- **普通用户**：企业资料、我的商机、对接待办、发布与对接
- **运营账号**：运营工作台、待审中心、代发管理；不能以个人身份发布或申请对接
- 「下一步待办」：普通用户的跨模块动作提示
- **对接待处理提醒**：普通用户看个人对接；运营账号看审核与代发待办

## 数据说明

支持 **本地演示** 与 **云开发多人模式** 两种运行方式。

### 本地模式（默认）

`utils/config.js` 中 `useCloud: false`，数据存在手机本地，适合单人体验。

### 云开发模式（多人）

设置 `useCloud: true` 并填写 `cloudEnvId` 后，数据存入云数据库，多用户共享资源池、需求池与对接流程。

详细部署步骤见 [09_云开发部署指南.md](09_云开发部署指南.md)。

### 本地缓存 key（云端模式下作为同步缓存）

| Key | 说明 |
| --- | --- |
| `compute_trade_published_resources` | 用户发布的资源 |
| `compute_trade_published_demands` | 用户发布的需求 |
| `compute_trade_submissions` | 提交记录 |
| `compute_trade_user_profile` | 企业资料 |
| `compute_trade_platform_init` | 空白模式版本标记 |

云端模式（`useCloud: true`）下，以上 key 仅作本机缓存；**所有读写以云数据库为准**（`users` / `listings` / `submissions` 三个集合，经 `tradeApi` 云函数）。收藏列表存在 `users.favoriteResources` / `users.favoriteDemands`。页面打开时会自动 `sync` 拉取最新数据；收藏在登录后跨设备同步。本地演示模式（`useCloud: false`）仍可在「我的」页底部清除。
